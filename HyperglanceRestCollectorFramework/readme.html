Readme for Hyperglance Rest Framework Merged Collector

 

This version of the collectors runs as a unifide merged process. Many collectors can be set up using a discovery 'self registration' mechanisiam. It is initially configured with all four collectors, but if one or more are not required then simply delete the associated directory. The order of the collectors can be set by the xxxx.execute.order= value

Getting Started

  1. Unpack the supplied jar
  2. Edit the launch-hyperglance-rest-framework.bat or launch-hyperglance-rest-framework.sh file, and set a JAVA_HOME to point to a JRE or a JDK.
  3. Edit the hyperglance-rest-framework.properties file to configure the connection to the Hyperglance Rest API. See notes in the file for detailed information.
  4. Remove the collectors that are not required by simply deleting the appropriate directory. (NOTE: There will still be a reference on the classpath in launch-hyperglance-rest-framework.bat or launch-hyperglance-rest-framework.sh but that will just be ignored.
  5. Edit the property file in each of the individual collectors to provide the collector specific properties.
    • Take care if cutting and pasting from a previous version as they can change every version.
    • Each collector can set the individual collector name that is placed into the 'origin' field
    • Each collector can have an execute order set through xxxx.execute.order= value. The collectors are then executed in ascending order. This becomes important when you have refinement collectors that need to work with the whole model.
  6. NOTE: The VMWare instance also has the standalone credentials app that uses the same propeties file.
  7. Launch either the launch-hyperglance-rest-framework.bat or launch-hyperglance-rest-framework.sh file
